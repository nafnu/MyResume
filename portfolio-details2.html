About AWS CloudFormation Project

The project focuses on deploying a high-availability web application named Udagram using AWS CloudFormation. It is a hands-on implementation designed to demonstrate Infrastructure as Code (IaC) principles.

The main goal is to create an automated, scalable, and resilient infrastructure by provisioning resources in AWS through CloudFormation templates.



Aspects important during the development:

  • CloudFormation Templates: YAML and JSON files are used to define and provision AWS resources programmatically.
  • Scripts: Helper scripts (create.sh, update.sh) are included for stack creation and updates to simplify AWS CLI commands.
  • Testing: Tools like cfn-lint are recommended to validate CloudFormation templates, ensuring compliance with AWS best practices.


The next diagram representation is the infrastructure diagrams illustrate the architecture, showing key components and their relationships.



Features

  • Infrastructure Automation: Uses CloudFormation templates to set up the required AWS resources such as EC2 instances, Load Balancers, Auto Scaling groups, and RDS databases.
  • High Availability: Implements redundancy and fault tolerance by distributing resources across multiple Availability Zones.
  • Security Best Practices: Includes a Virtual Private Cloud (VPC) setup with public and private subnets, security groups, and IAM roles.
  • Customizable Parameters: Templates allow users to specify configurations such as instance types, key pairs, and network settings.

It is so handy to do auto Scaling groups dynamically adjust the number of EC2 instances based on traffic patterns.
